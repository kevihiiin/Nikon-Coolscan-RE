# SCSI Sense Code Catalog — Nikon LS-50/LS-5000

**Status**: Complete
**Last Updated**: 2026-02-28
**Phase**: 5 (Protocol Spec)
**Confidence**: Verified (decoded from firmware sense translation table at 0x16DEE + handler cross-references)

## Overview

The LS-50 firmware uses a two-level sense code system:

1. **Internal sense index** (16-bit word at RAM `0x4007B0`): Set by SCSI command handlers when an error occurs.
2. **Sense translation table** (148 entries at flash `0x16DEE`): Each entry is 5 bytes `[Flags, SenseKey, ASC, ASCQ, FRU]`. The REQUEST SENSE handler at `0x021866` calls subroutine `0x0111F4` to translate the internal index into a standard 18-byte SCSI fixed-format sense response.

### Sense Response Format (18 bytes)

Built by subroutine at `firmware:0x0111F4`:

| Byte | Field | Source |
|------|-------|--------|
| 0 | Response Code | `0x70` (current) or `0xF0` (deferred, if flags bit 7 set) |
| 1 | Segment Number | 0 |
| 2 | Sense Key | From table, OR'd with `0x20` if flags bit 4 set (ILI) |
| 3-6 | Information | From RAM `0x4007A0` (if flags bit 5 set) |
| 7 | Additional Sense Length | `0x0B` (11 bytes follow) |
| 8-11 | Command-Specific Info | 0 |
| 12 | ASC (Additional Sense Code) | From table |
| 13 | ASCQ (Additional Sense Code Qualifier) | From table |
| 14-15 | Sense Key Specific | From RAM `0x4007A4` (if flags bit 3 set) |
| 16 | FRU (Field Replaceable Unit) | From table |
| 17 | Reserved | 0 |

### Translation Table Entry Format (5 bytes at 0x16DEE + index × 5)

| Offset | Field | Description |
|--------|-------|-------------|
| 0 | Flags | Bit 7: deferred (0xF0 response code); Bit 5: include info bytes; Bit 4: ILI bit; Bit 3: include SKSV bytes |
| 1 | Sense Key | Standard SCSI sense key (0x00-0x0B) |
| 2 | ASC | Additional Sense Code |
| 3 | ASCQ | Additional Sense Code Qualifier |
| 4 | FRU | Field Replaceable Unit code |

## Sense Codes by Sense Key

### SK=0 (NO SENSE)

| Index | ASC/ASCQ | Meaning | Generated By |
|-------|----------|---------|-------------|
| 0x00 | 00/00 | No error | Cleared after successful commands |
| 0x01 | 00/00 | No error (deferred+ILI) | GET WINDOW, SCAN |
| 0x02 | 00/00 | No error (deferred) | SCAN |
| 0x03 | 00/00 | No error (deferred) | Shared module |

### SK=1 (RECOVERED ERROR)

| Index | ASC/ASCQ | FRU | Meaning | Generated By |
|-------|----------|-----|---------|-------------|
| 0x04 | 37/00 | 00 | Rounded parameter | GET WINDOW, SCAN, SEND DIAGNOSTIC |
| 0x05 | 61/02 | 01 | Out of focus (channel 1) | INQUIRY |
| 0x06 | 61/02 | 02 | Out of focus (channel 2) | INQUIRY, READ |

*Note*: ASC 37h = "Rounded Parameter" means the scanner adjusted a requested value (e.g., resolution) to the nearest supported value. ASC 61h/ASCQ 02h = "Out of focus" with FRU indicating which CCD channel detected the condition.

### SK=2 (NOT READY)

| Index | ASC/ASCQ | FRU | Meaning | Generated By |
|-------|----------|-----|---------|-------------|
| 0x07 | 04/01 | 00 | Becoming ready (general) | TEST UNIT READY, INQUIRY |
| 0x08 | 04/01 | 01 | Becoming ready (USB/ISP1581) | TEST UNIT READY, INQUIRY |
| 0x09 | 04/01 | 02 | Becoming ready (encoder) | TEST UNIT READY, SCAN |
| 0x0A | 04/02 | 00 | Initialization command required | READ, SEND DIAGNOSTIC |
| 0x0B | 04/03 | 00 | Manual intervention required | INQUIRY |
| 0x0D | 05/00 | 00 | LU does not respond (ejecting) | TEST UNIT READY |
| 0x0E | 3A/00 | 00 | Medium not present | SCAN |
| 0x0F | 3A/00 | 01 | Medium not present (variant) | INQUIRY |
| 0x79 | 04/01 | 03 | Motor busy (positioning) | TEST UNIT READY |
| 0x71 | 04/02 | 00 | Scan timeout (reinit required) | TEST UNIT READY |
| 0x7A | 04/01 | 04 | Calibration in progress | TEST UNIT READY |
| 0x80 | 3A/00 | 02 | Medium not present (adapter) | Shared module |
| 0x82 | 3A/00 | 04 | Medium not present (type 4) | Shared module |

*Note*: The "Becoming ready" variants (04/01) use FRU to distinguish the cause: 00=general startup, 01=USB controller initializing, 02=film encoder calibrating, 03=motor positioning, 04=lamp/CCD calibration. The host should retry TEST UNIT READY. "Medium not present" (3A/00) means no film adapter or no film loaded.

### SK=3 (MEDIUM ERROR)

| Index | ASC/ASCQ | FRU | Meaning | Generated By |
|-------|----------|-----|---------|-------------|
| 0x10 | 11/00 | 00 | Unrecovered read error (general) | Shared module |
| 0x11 | 11/00 | 10 | Read error (channel 1) | INQUIRY |
| 0x12 | 11/00 | 20 | Read error (channel 2) | SCAN |
| 0x14 | 11/00 | 90 | Read error (multi-channel) | Shared module |
| 0x16 | 11/00 | 11 | Read error (ch1, deferred) | SCAN |
| 0x18 | 11/00 | 31 | Read error (ch3, deferred) | Shared module |
| 0x1C | 11/00 | 22 | Read error (ch2, deferred) | INQUIRY |
| 0x1E | 11/00 | 92 | Read error (multi, deferred) | INQUIRY |

*Note*: FRU encodes the CCD channel (high nibble: 1=R, 2=G, 3=B, 9=multi) and failure subtype (low nibble: 0=current, 1=deferred with info bytes, 2=deferred without info).

### SK=4 (HARDWARE ERROR)

| Index | ASC/ASCQ | FRU | Meaning | Generated By |
|-------|----------|-----|---------|-------------|
| 0x20 | 03/00 | 01 | Write fault (data path) | Shared module |
| 0x21 | 08/01 | 00 | Communication timeout | INQUIRY |
| 0x27 | 11/01 | 91 | Read retries exhausted | INQUIRY |
| 0x2E | 44/00 | 01 | Internal target failure | SCAN, SET WINDOW |
| 0x35 | 60/00 | 21 | Lamp failure (G ch, type 1) | SET WINDOW |
| 0x38 | 60/00 | 02 | Lamp failure (R ch, type 2) | Shared module |
| 0x3A | 60/00 | 22 | Lamp failure (G ch, type 2) | GET WINDOW |
| 0x40 | 60/00 | 33 | Lamp failure (B ch, type 3) | Shared module |
| 0x44 | 60/00 | 24 | Lamp failure (G ch, type 4) | Shared module |
| 0x72 | 4C/00 | 01 | Self-configuration failed | Shared module |
| 0x84 | 53/00 | 04 | Media load/eject failed | Shared module |
| 0x85 | 60/00 | 1A | Lamp failure (R ch, type A) | Shared module |
| 0x88 | 60/00 | 9A | Lamp failure (multi, type A) | Shared module |
| 0x8D | 60/00 | 19 | Lamp failure (R ch, type 9) | Shared module |

*Note*: The firmware defines **30 total lamp failure entries** (indices 0x33-0x4B, 0x85-0x90) — the largest single group. FRU encodes: high nibble = channel (0=R, 1=G, 2=B, 3=IR, 9=multi), low nibble = failure sub-type. ASC 44h = "Internal target failure" is the general hardware error code. ASC 53h = "Media load/eject failed" indicates motor failure during film transport.

### SK=5 (ILLEGAL REQUEST)

| Index | ASC/ASCQ | Meaning | Generated By |
|-------|----------|---------|-------------|
| 0x4E | 1A/00 | Parameter list length error | MODE SELECT, RECEIVE DIAGNOSTIC, SEND DIAGNOSTIC |
| 0x4F | 20/00 | Invalid command operation code | Command dispatch (unrecognized opcode) |
| 0x50 | 24/00 | Invalid field in CDB | All handlers (most common error) |
| 0x51 | 25/00 | Logical unit not supported | Command dispatch (LUN != 0) |
| 0x52 | 25/00 | LUN not supported (variant) | REQUEST SENSE |
| 0x53 | 26/00 | Invalid field in parameter list | MODE SELECT, SCAN, SEND DIAG, SET WINDOW, VENDOR E0 |
| 0x54 | 26/01 | Parameter not supported | Shared module |
| 0x55 | 26/02 | Parameter value invalid | SCAN |
| 0x56 | 2C/00 | Command sequence error | Dispatch, READ, RECEIVE DIAG, VENDOR C1 |
| 0x57 | 2C/01 | Too many windows | INQUIRY |
| 0x58 | 2C/02 | Invalid window combination | SCAN |
| 0x59 | 39/00 | Saving parameters not supported | MODE SENSE |
| 0x5A | 3D/00 | Invalid bits in identify message | INQUIRY |

*Note*: The most common sense code is 0x50 (ASC 24h = "Invalid field in CDB"), set by virtually every handler when reserved CDB bytes are non-zero or a parameter value is out of range. ASC 20h = invalid opcode is only set by the main dispatch table when an opcode has no handler. ASC 2Ch = "Command sequence error" typically means a command was issued out of order (e.g., READ before SET WINDOW+SCAN, or C1 before E0).

### SK=6 (UNIT ATTENTION)

| Index | ASC/ASCQ | Meaning | Generated By |
|-------|----------|---------|-------------|
| 0x5C | 29/00 | Power on/reset/bus device reset | Shared module (after USB bus reset) |
| 0x61 | 3F/03 | Inquiry data has changed | INQUIRY (adapter change detected) |

*Note*: The host must clear UNIT ATTENTION by issuing any command. ASC 29h is set after power-on or USB bus reset. ASC 3Fh/ASCQ 03h indicates that the film adapter type changed (e.g., strip holder swapped for slide mount).

### SK=9 (VENDOR SPECIFIC)

| Index | ASC/ASCQ | FRU | Meaning | Generated By |
|-------|----------|-----|---------|-------------|
| 0x62 | 80/01 | 02 | Vendor hardware issue | SCAN |
| 0x64 | 80/02 | 00 | Vendor scan data issue | SCAN |
| 0x81 | 80/01 | 06 | Motor control error | SCAN |
| 0x92 | 80/06 | 01 | Scan parameter error | SCAN |

*Note*: All vendor-specific sense codes use ASC 80h (in the vendor-specific range). They are generated exclusively by scan operations when internal hardware or data integrity checks fail.

### SK=B (ABORTED COMMAND)

| Index | ASC/ASCQ | Meaning | Generated By |
|-------|----------|---------|-------------|
| 0x65 | 08/00 | LU communication failure | Dispatch, READ, SEND DIAGNOSTIC |
| 0x66 | 3E/00 | LU has not self-configured yet | Dispatch |
| 0x68 | 45/00 | Select/reselect failure | Dispatch |
| 0x6F | 4B/00 | Data phase error | Dispatch, READ |

*Note*: These indicate transport-level or initialization failures. ASC 08h typically means the ASIC or DMA engine failed to respond. ASC 3Eh means the firmware hasn't completed its initialization sequence yet. ASC 4Bh indicates a data transfer size mismatch.

## Host-Side Error Handling

NikonScan4.ds maps MAID error codes to user-visible messages:

| MAID Error | Value | Sense Trigger | UI Action |
|------------|-------|---------------|-----------|
| BUSY | -0x7A (-122) | SK=2, ASC 04/01 | Shows "scanner busy", retries |
| TIMEOUT | -0x76 (-118) | SK=4, ASC 08/01 | Shows "timeout" |
| CANCELLED | -0x7B (-123) | User-initiated abort | Cancels queue |
| SPECIFIC_ERROR | 0x182 (386) | Various HW errors | Shows error dialog |

LS5000.md3 sense handling (at `LS5000.md3:0x100ae3c0`): After each SCSI command execution, the driver retrieves sense data via the REQUEST SENSE mechanism (0x06 on bulk-out → sense data on bulk-in). Non-zero sense keys trigger error callbacks through the MAID interface.

Source: `NikonScan4.ds:0x1004d0c0` (StatusHandler), `LS5000.md3:0x100ae3c0` (command executor)
